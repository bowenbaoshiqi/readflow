# v0.6 阅读器双 Bug 修复 TDD 证据报告

**日期**: 2026-07-16
**分支**: feat/v0.4-ai-brains
**提交**: c67325d (Bug1), da86f1b (Bug2)

## 背景

v0.6 解决两个阅读器 bug:

1. **Bug1**: 看书时底部出现红色 error 条,报 `foliate-js/paginator.js 806:36` 和 `862:26` TypeError
2. **Bug2**: 选中文字时弹出的"划线/复制"按钮遮挡正文(尤其选中靠近底部的文字)

## 根因分析

### Bug1: foliate-js 上游边界崩溃

`paginator.js` 两个私有字段声明时无初值(`undefined`):

- `#scrollBounds`(line 448)— 仅在 `#scrollTo` 内部赋值(895/905/910)
- `#touchState`(line 449)— 仅在 `#onTouchStart` 内部赋值(825)

边界场景下解构/读取崩溃:

| 报错位置 | 代码 | 触发条件 |
|---------|------|---------|
| `862:26` | `this.snap(this.#touchState.vx, …)` | `touchend` 无前置 `touchstart` → `#touchState` undefined |
| `806:36` | `const [offset, a, b] = this.#scrollBounds` | 首帧 `#scrollTo` 未跑完即滑动 → `#scrollBounds` undefined |

调用链: `#onTouchEnd` (853) → `requestAnimationFrame` (860) → `this.snap(...)` (862) → 解构 `#scrollBounds` (806)

**关键**: foliate-js submodule HEAD `78914ae` 正是上游 `heads/main` 最新,无更新版可升。该崩溃是上游 bug,非我方版本旧。

`reader.py:113-126` 的全局 error handler 把这个本应静默的异常画成 `position:fixed;bottom:0;z-index:999` 红色 `<pre>` 横条,放大了可见性。

### Bug2: 底部固定浮层遮挡

`#bottom-bar` 样式(`reader.css:25-31`):

```css
#bottom-bar {
  position: fixed; bottom: 16px; left: 50%; transform: translateX(-50%);
  z-index: 20;
}
```

固定钉在视口底部居中。foliate iframe 正文铺满到底,用户选中靠近底部的文字(读书最常见)→ `reader.js` 轮询检测到选区 → 工具条弹出 → 正好压在选区上。

## 方案(用户选定)

### Bug1: Dockerfile 层 patch(方案 C)

补丁不能直接改 submodule 工作树(`git submodule update --init` 会冲掉)。采用 Dockerfile 层 patch:

- 补丁以独立文件 `patches/foliate-paginator.patch` 存仓库
- Dockerfile build 阶段 `patch -p1` 应用到 foliate-js/paginator.js
- submodule 工作树保持上游原样

三处 guard:
```diff
+ if (!this.#scrollBounds) return  // scrollBy / snap 入口
+ if (!this.#touchState) return    // #onTouchEnd 的 rAF 回调入口
```

> 备选方案 A(vendor 化)未选:脱离上游 git 历史,未来升级需手动 diff。
> 备选方案 B(子模块内提交)排除:fork 无自有 remote,clone 后 submodule init 会 hash 对不上。

### Bug2: 按钮移到顶部 toolbar(方案 C)

移除 `#bottom-bar`,划线/复制按钮进 `#toolbar`,默认 `hidden`,选中文字才显示。

## 已知局限

- Bug1 补丁依赖 Dockerfile build 阶段 `patch -p1`;本地非 Docker 跑 uvicorn 时 paginator.js 是上游原版(无补丁),本地复现不到该崩溃。开发环境若需复现可手动 `git apply --directory=app/static/foliate-js patches/foliate-paginator.patch`。
- Bug1 的 TypeError 在 Kindle 老内核触摸行为下更易触发,补丁使其静默 return,但 snap 动画该次会丢失(不影响翻页功能)。
- foliate-js 未来若升级上游,`patches/foliate-paginator.patch` 可能因上下文偏移 apply 失败 —— Dockerfile build 会即时报错(不会静默回归),届时需重新生成补丁。
