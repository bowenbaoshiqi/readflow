# TDD 证据报告 — v0.4 阅读页返回按钮回书架

## 1. 来源

本次 TDD 运行中派生的需求(无 `*.plan.md` 输入)。

## 2. 用户 journey

> 作为读者,我想在阅读页点"返回"按钮直接回到书架首页,这样不管我是从哪里进来的(书架直进 / 详情页转进 / 直接打开链接),返回行为都一致且可靠。

## 3. 问题

`app/static/reader.js:143` 原实现:

```js
document.getElementById('back').onclick = () => history.back()
```

`history.back()` 是浏览器历史回退,行为不确定:

- 从 **书架 → 详情页 → 阅读页** 进来时,`back()` 只退到详情页,不是书架
- 直接打开阅读页链接(无历史)时,`back()` 卡住,按钮无反应

用户预期"返回" = 回书架首页 `/`。

Kindle 路径 `/read-html/{id}` 的返回是 `<a href="/">←书库</a>`,本身正确,本次不动。
