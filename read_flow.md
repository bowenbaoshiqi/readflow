## 一、产品定位

**一句话：** 部署在 Synology DS220+ 上的私人阅读服务，管理 NAS 本地书库，提供在线阅读 + AI 驱动的每日总结、个性化推荐。

**不是什么：** 不是电子书下载站、不是社交阅读平台、不是 Calibre 替代品。

**核心价值主张：** 书放在你自己硬盘里，但 AI 帮你读得更深、发现更多。

---

## 二、目标用户

一个人：t91。单用户系统，无多账号、无权限管理。

## 三、业务模块

### 3.1 书籍管理

| 能力 | 说明 |
|---|---|
| 文件监听 | 监听 NAS 指定书库目录，新文件自动入库 |
| 支持格式 | epub、pdf、mobi、azw3、txt |
| 格式转换 | 非 epub 格式入库时自动转换为 epub（Calibre `ebook-convert` 命令行），统一后续渲染管线；原文件保留不动 |
| 元数据提取 | 从 epub（或转换后的 epub）提取书名、作者、封面、总页数 |
| 豆瓣摘要 | 自动搜索豆瓣，拉取简介（≤200字）、评分、标签 |
| 降级搜索 | 豆瓣失败时，用搜索引擎搜"书名+简介"，取前 200 字 |
| 全文搜索 | 入库后建立 Meilisearch 索引，支持按书名和内容搜索 |
| 存储策略 | 原始文件 + 转换后的 epub 均留在 NAS 目录，服务只记录路径和元数据 |

**格式转换说明：**

```
新文件入库 → 判断格式
  ├─ epub → 直接解析
  ├─ pdf → ebook-convert → epub → 解析
  ├─ mobi → ebook-convert → epub → 解析
  ├─ azw3 → ebook-convert → epub → 解析
  └─ txt → ebook-convert → epub → 解析
```

Calibre 的 `ebook-convert` 命令行工具通过 Docker 内安装 calibre 包获得，Python 通过 subprocess 调用。转换是异步任务——新书入库后先展示元数据，转换完成后更新状态。

### 3.2 读书服务

| 能力 | 说明 |
|---|---|
| 在线阅读器 | 浏览器内渲染 epub（分章节 HTML），支持暗色模式 |
| 排版设置 | 字号(12-28px)、行距(1.2-2.8)、页边距(窄/中/宽)、字体选择，localStorage 持久化 |
| 阅读定位 | spine_index + offset_percent，浮动页码无关，跨设备精准 |
| 热书缓存 | 最多缓存 10 本书的解析内容；超出时用户选择剔除；冷书实时解析并加入缓存 |
| 划线 | 选中文字划线，存入 highlight 记录 |
| 原始文本记录 | 每翻一页记录该页纯文本全文，含时间戳和划线标记 |
| 阅读进度 | 后台记录每本书上次阅读位置 |

### 3.3 每日总结

| 能力 | 说明 |
|---|---|
| 触发时机 | 每日凌晨定时任务 |
| 数据来源 | 过去 24 小时的 reading_log，按书聚合 |
| AI 处理 | 调 OpenAI API，生成每本书的当日内容摘要（200-300字） |
| 展示位置 | 首页"昨日阅读总结"卡片 |

### 3.4 推荐系统

每日更新，首页展示 20 本构成：

| 数量 | 类别 | 推荐语 |
|---|---|---|
| 15 本 | 主题相似推荐 | AI 基于最近 7 天阅读内容生成个性化推荐语，解释"为什么你现在应该读这本" |
| 5 本 | 随机探索 | 书库中冷门但评分尚可，展示豆瓣摘要，不生成 AI 推荐语 |

### 3.5 AI 集成

| 层面 | 决策 |
|---|---|
| 提供商 | 用户提供 API key（OpenAI 兼容接口） |
| 配置方式 | 后端环境变量 |
| 调用策略 | 批量调用优先（每日凌晨一次汇总处理），结果缓存（摘要不重复生成） |
| 降级 | AI 不可用时，阅读、搜索、位置记录全部不受影响 |

## 四、前端页面

### 4.1 首页 — "今日"

- 昨日阅读总结卡片（按书聚合，显示阅读页数和 AI 摘要）
- 今日推荐瀑布流（20 本，前 15 带个性化推荐语，后 5 带豆瓣摘要）
- 右上角设置入口（仅排版设置）

### 4.2 书库 — "书库"

- 搜索栏（书名 + 全文搜索）
- 网格布局书籍卡片（封面、书名、作者、阅读进度、豆瓣评分）
- 排序切换（最近添加 / 最近阅读 / 评分）
- 响应式：桌面 4-5 列，平板 3 列，手机 1-2 列

### 4.3 阅读器 — "/read/{id}"

- 章节分页渲染，上下翻页或滚动模式
- 顶部工具栏：返回、书名/章节名、进度百分比、排版设置
- 底部工具栏（选中文字触发）：划线、复制、AI 解释、翻译
- 排版设置面板：字号、行距、页边距、字体

## 五、技术架构

| 层面 | 选型 | 理由 |
|---|---|---|
| 运行环境 | Docker 单容器，部署于 DS220+ | J4025 2C/2T，单容器最小化 |
| 后端框架 | Python FastAPI | ebooklib、httpx、openai SDK 生态 |
| 数据库 | SQLite | 零维护，足够 |
| 搜索引擎 | Meilisearch | 低内存、中文分词 |
| 格式转换 | Calibre `ebook-convert` | 命令行调用，格式覆盖最全 |
| 前端 | 原生 HTML/CSS/JS（Alpine + htmx） | 低依赖 |
| 定时任务 | APScheduler | Python 原生 |
| 文件监听 | watchdog | Python inotify 封装 |
| AI SDK | openai Python SDK | 兼容 OpenAI API |
| 部署 | Dockerfile + docker-compose | 一键启动 |

**资源预估（DS220+ 实际负载）：**

| 指标 | 估值 |
|---|---|
| 容器内存 | 400-600MB（含 Calibre） |
| 存储 | 数据库几百 MB + 转换后 epub 与原文件同目录 |
| CPU（日常） | 几乎空闲 |
| CPU（格式转换时） | 短暂升高，一本 PDF 转 epub 几秒到几十秒 |
| CPU（凌晨批处理） | 短暂峰值，AI 调用在云端 |

## 六、非功能需求

| 维度 | 要求 |
|---|---|
| 隐私 | 所有数据和文件在本地，AI 调用仅发送当日阅读文本摘要 |
| 离线降级 | AI 不可用时阅读和搜索不受影响 |
| 响应式 | 桌面和移动均可正常阅读 |
| 持久化 | 阅读位置、排版、划线持久化 |

## 七、不做的事（v0.1 边界）

- 不做多用户
- 不做 Kindle 推送
- 不做阅读统计/阅读报告页面
- 不做笔记导出
- 不做 PWA 离线缓存
- 不做渐进式回顾（已从推荐中移除）
- 不做 AI 对话式问答（先做选中文字的解释和翻译）

## 八、风险与应对

| 风险 | 应对 |
|---|---|
| 豆瓣反爬 | 降级搜索引擎搜索，速率控制，user-agent 伪装 |
| DS220+ 性能瓶颈 | 单容器设计，Meilisearch 替代 ES，ebook-convert 异步执行 |
| epub 格式兼容性 | ebooklib 覆盖主流 epub，异常跳过并日志告警 |
| 格式转换失败 | 异步任务状态标记，转换失败的书标记为"不受支持"，用户可见 |
| AI API 费用 | 每日约 25-35 次调用，结果缓存避免重复 |
